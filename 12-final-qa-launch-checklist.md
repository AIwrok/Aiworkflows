# 12 — Final QA and Launch Checklist

## Visual QA

Check every section:

- Header looks premium.
- Logo is clear and not too large.
- Navbar is readable.
- Hero headline is strong.
- CTA buttons are visible.
- Cards have consistent style.
- Spacing feels premium.
- Sections do not look crowded.
- Gradient usage is controlled.
- Footer looks complete.

## Content QA

Check:

- No spelling mistakes.
- No broken English in main headings.
- Services are clear.
- Industries are clear.
- Projects have CTAs.
- Pricing is not confusing.
- Contact action is clear.
- No placeholder text remains.

## Mobile QA

Test screen widths:

- 360px
- 390px
- 414px
- 768px
- 1024px
- Desktop

Check:

- No horizontal scrolling.
- Mobile menu works.
- Buttons are easy to tap.
- Text does not overflow.
- Cards stack properly.
- Hero cards do not break layout.
- Pricing section is readable.
- Contact form/buttons are usable.

## Animation QA

Check:

- Animations are smooth.
- No animation feels childish.
- No section appears too late.
- No layout shift caused by animation.
- Reduced motion is supported.
- Mobile performance is good.

## Link QA

Check all navigation links:

- Services
- Industries
- Demos
- Projects
- Proof
- Process
- Team
- Pricing
- Contact
- Book Free Demo

Check all project/demo links.

## SEO QA

Check:

- One H1 only.
- All sections use proper H2.
- Meta title added.
- Meta description added.
- Open Graph title/image if possible.
- Images have alt text.
- Local business keywords included naturally.

## Performance QA

Run:

```bash
npm run build
npm run start
```

Check browser console:

- No red errors.
- No hydration mismatch.
- No missing images.
- No broken imports.

Use Lighthouse:

- Performance 85+
- Accessibility 95+
- Best Practices 95+
- SEO 90+

## Client conversion QA

Ask these questions:

1. Can a visitor understand the business in 5 seconds?
2. Can a restaurant owner understand what to click?
3. Can a cyber café owner see relevant solution?
4. Can a distributor see relevant software value?
5. Can a visitor book a demo easily?
6. Does the website show proof?
7. Does pricing create trust?
8. Does mobile experience feel professional?

## Final launch steps

1. Backup current version.
2. Deploy to staging if available.
3. Test on mobile.
4. Test all links.
5. Share with team.
6. Collect feedback.
7. Fix final issues.
8. Deploy production.
9. Announce on LinkedIn.
10. Start using website in client pitches.
